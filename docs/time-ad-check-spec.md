# 의료광고 자가검수 도구 (/time/ad-check) — 설계 스펙

> 발주: 열정의시간(대표 한동남). 상위 인수인계서: `passion-intranet/의료광고검수 프로그램/의료광고검수도구_인수인계서_v1.md`
> 이 문서는 홈페이지(passion-homepage) 공개 MVP 범위의 단일 소스다.

## 1. 목적 · 퍼널

원장·실무자가 블로그·이벤트·SNS 문구를 붙여넣으면 의료법 저촉 문장을 신호등으로 판정하고, 근거 조항과 바로 쓰는 수정문안을 제시하는 **무료 자가검수 도구**. 결과 아래 담백한 회사 소개 + 카카오톡 채널 문의로 자연 유입을 만든다(하드셀 금지).

퍼널: 문구 입력(무로그인) → 신호등·하이라이트·근거·수정문안 → 회사 소개 + 카톡 문의(`pf.kakao.com/_RgYcxj/chat`).

## 2. 아키텍처 — 엔진 1개, 껍데기

| 계층 | 파일 | 역할 |
|---|---|---|
| 룰 사전 | `src/lib/ad-review/rules.ts` | 의료법 56조2항 14개 호 + 27조3항 키워드·정규식·수정문안·허용조건 (정답 데이터, 코드 상수) |
| 검수 엔진 | `src/lib/ad-review/engine.ts` | `runRuleScan(text)` 순수 함수 — 매칭·위험도·하이라이트 span·7호 누락 판정 |
| API | `src/app/api/ad-check/route.ts` | POST. 1단 룰베이스(항상) + 2단 Claude 문맥판정(선택, 실패 시 룰베이스 폴백) |
| 페이지 | `src/app/time/ad-check/{page.tsx,layout.tsx,ad-check.css}` | 입력 UI·결과 렌더·회사소개·카톡. 딥크림슨+샴페인골드 톤 |

**핵심 원리**: AI가 법을 외워 판단하지 않는다. 사람이 정리한 법(rules.ts)에 대조한다. Claude 호출 시에도 규칙 요약을 시스템 프롬프트에 주입해 근거를 고정한다.

## 3. 판정 로직

1. **1단 룰베이스**: 14호 키워드/정규식 매칭 → 위험도(high/medium/gray) → 하이라이트 span 병합 → 위반 카드(근거조항·사유·허용조건·수정문안). 7호(부작용 누락)는 시술 언급 + 부작용 고지 부재로 감지.
2. **2단 Claude 문맥판정**: 회색지대(대가성 후기·명칭·은근한 과장·비교)를 규칙 기준 대조로 판정, JSON 반환. `ANTHROPIC_API_KEY` 없거나 실패 시 null 폴백(룰베이스만 노출).
3. 신호등: high=위험(빨강), medium/gray=주의(노랑), 없음=안전(초록).

## 4. MVP 범위 · 제외

- 포함: 텍스트 검수, 신호등, 문장 하이라이트, 근거조항, 수정문안, 면책, 회사소개+카톡, 룰+AI 2단.
- 제외(다음): 이미지/URL·OCR, PDF 리포트, 매체선택 심의대상 판정, 검수 로그 DB(전환분석), 인트라넷 내부용 화면, 크롤링 파이프라인, 외부 SaaS 분리·RLS.

- 노출: 초기 `robots: noindex` + `/time` 네비 미노출(조용한 URL 테스트). 반응 확인 후 색인·메뉴 노출로 전환.

## 6. 준수

- YMYL: 결과에 면책 문구 상시 노출. "변호사 대체" 아님, 참고 도구.
- 룰 사전 기준선·수정문안은 사람(법무) 검토로 정확도 담보. 실전 반려사례 누적 시 고도화.
- 회색지대(전후사진 5요건·비급여 할인)는 차단이 아니라 통과 조건 안내.
